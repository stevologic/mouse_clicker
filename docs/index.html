Everything a clicker should be

Extremely configurable

Dial in exactly the behavior you need — from a blistering 120 CPS to a slow, human-looking idle jiggle.

⟳

Every click type

Left, right, middle, or side buttons. Single, double, triple, N-click bursts, scroll, or press-and-hold — with randomized hold times that read as human.

⏱

Precise timing

Fixed or randomized intervals, quick CPS presets, a start countdown, and run limits by click count or duration.

↗

Anywhere on screen

Click at the cursor, a fixed point, a random spot inside a region, or walk a looping sequence of points across every monitor.

〰

Humanized movement

Teleport, glide, or move along curved Bézier paths with easing and micro-jitter so the cursor looks hand-driven, not scripted.

●

Record & replay

Hit Record and use your mouse anywhere — mouseclicker.app captures the cursor movement, presses, and releases (so click-and-drag works), with full timing, then retraces the whole sequence a set number of times or on a loop.

⌨

Global hotkeys

Start, stop, and emergency-stop from anywhere with system-wide hotkeys — even while mouseclicker.app sits in the background.

✦

AI patterns, any model

Describe what you want in plain English and let Claude, OpenAI, Gemini, Grok, or a lightweight local model build the whole pattern. Detect and download local models right inside the app and run them fully offline via Ollama — no key, no cloud — or fall back to the built-in generator.

◎

Live activity HUD

While a run is active, a sleek floating popup shows a pulsing indicator and a live click counter — always on top, and click-through so it never gets in the way.

✧

Living interface

A hand-built animated UI — a cursor-reactive particle constellation, an aurora backdrop, glassmorphism, and a glowing Start button. It idles to near-zero CPU when in the background.

Claude · OpenAI · Gemini · Grok · Local

Just describe it

Type what you want in plain English. Pick your provider — Claude, OpenAI, Gemini, Grok, or a local model via Ollama — and mouseclicker.app translates it into a precise, ready-to-run pattern.

"Click like a human every 1–3 seconds near the center of the screen, with slight random movement, for 5 minutes."

The local option needs no key and no internet — a tiny model like qwen2.5:0.5b runs right on your machine. The app detects your installed models and can download a new one for you, all in-app. No Ollama? A built-in offline generator and one-click presets have you covered. Cloud keys are stored locally and sent only to the provider you choose.

Get started

Running in 30 seconds

Nothing to install. Download the executable and run it — or build it yourself from source.

Download

Grab MouseClicker.exe from the latest release.

Run it

Double-click. It runs on the .NET Framework already built into Windows — no runtime download.

Configure or ask AI

Set your options, or open the AI tab and describe the pattern you want.

Press your hotkey

Hit F6 to start and stop from anywhere. F8 is the emergency stop.

Prefer to build from source? You only need Windows — the compiler ships with it.

git clone https://github.com/stevologic/mouse_clicker
cd mouse_clicker
powershell -ExecutionPolicy Bypass -File build.ps1 -Run
Trust & safety

“Windows protected your PC”?

That SmartScreen prompt is not a virus warning — Microsoft Defender scans mouseclicker.app clean. It shows up for any unsigned app that hasn't built download reputation yet (signing certificates cost money; this is a free, open-source project). Here's how to run it with confidence:

Build it yourself

The full source is here and compiles with the C# compiler already in Windows. A locally built exe shows no prompt at all. See Get started.

Verify the download

Every release lists a SHA-256. Check it: Get-FileHash .\MouseClicker.exe -Algorithm SHA256.

Run past the prompt

Click More info → Run anyway, or right-click the exe → Properties → tick Unblock → OK.

Because it synthesizes mouse input, some third-party antivirus tools may heuristically call it a “PUA / auto-clicker.” It only does what you configure — the source is public to audit, and any detection can be reported to your vendor as a false positive.

Building your own signed release? build.ps1 Authenticode-signs the exe automatically when you supply a code-signing certificate — no Windows SDK needed. See the signing guide.
